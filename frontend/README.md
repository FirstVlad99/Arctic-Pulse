# 📰 Пульс Арктики

## Описание проекта

Современная платформа для агрегации и отображения новостей с акцентом на арктическую тематику. Система собирает новости из различных источников, обрабатывает их с помощью AI, предоставляет удобный интерфейс для чтения, фильтрации и аналитики.

## 🚀 Технологический стек

### Frontend
- **React 18** - библиотека для построения пользовательских интерфейсов
- **Vite** - быстрый сборщик и инструмент разработки
- **React Router DOM v6** - маршрутизация
- **Ant Design** - компоненты интерфейса
- **TanStack React Query** - управление серверным состоянием
- **i18next** - интернационализация (русский/английский)
- **Recharts** - визуализация данных и графики
- **Day.js** - работа с датами

### Хранение на клиенте
- **IndexedDB** - кэширование пользовательских настроек и голосований

## ✨ Функциональные возможности

### 1. Главная страница (`/`)
- Отображение последних 7 новостей
- Показ самого свежего дайджеста (ежедневного/еженедельного)
- Адаптивный дизайн (десктоп/мобильный)
- Краткий обзор топ-новостей дайджеста

### 2. Каталог новостей (`/catalog`)
- **Поиск** по заголовкам и содержанию
- **Фильтрация** по:
  - Категориям (тегам)
  - Источникам
  - Периоду публикации
- **Пагинация** (20 новостей на страницу)
- Сортировка по дате публикации
- Лайки/дизлайки новостей (с сохранением в IndexedDB)

### 3. Просмотр новости (`/news/:id`)
- Полный текст с HTML форматированием
- Автоматическое открытие внешних ссылок в новой вкладке
- Отображение мета-информации (дата, источник, релевантность)
- Теги новости
- Возможность лайкнуть/дизлайкнуть
- Копирование ссылки на новость
- Краткое содержание (AI summary)

### 4. Дайджесты (`/digests`)
- Ежедневные и еженедельные дайджесты
- Фильтрация по дате
- Просмотр топ-новостей в дайджесте
- Детальный просмотр дайджеста с полным списком новостей

### 5. Аналитика (`/analytics`)
- Количество новостей по источникам (график)
- Распределение по темам (круговая диаграмма)
- Средняя надежность источников
- Средняя релевантность новостей
- Общее количество новостей в системе

### 6. Управление источниками (`/sources`)
- Просмотр всех источников новостей
- Индикатор надежности источника (в %)
- Статус системы (активен/отключен)
- Персональное управление (включение/выключение источников в ленте)

## 🌍 Интернационализация

Проект полностью переведен на два языка:
- 🇷🇺 Русский
- 🇬🇧 English

Переключение языка происходит через интерфейс (кнопка в шапке). Все даты форматируются в соответствии с выбранной локалью.

## 🎨 Адаптивный дизайн

- **Десктоп** (>1100px) - полный интерфейс с боковой панелью фильтров
- **Планшет** (768px - 1100px) - адаптированные отступы и размеры
- **Мобильный** (<768px) - компактный дизайн, выезжающие фильтры в Drawer, упрощенные карточки

## 🔧 Установка и запуск

### Требования
- Node.js 16+
- npm или yarn

### Установка

```bash
# Клонирование репозитория
git clone <repository-url>
cd <project-folder>

# Установка зависимостей
npm install

# Запуск в режиме разработки
npm run dev

# Сборка для production
npm run build

# Предпросмотр production сборки
npm run preview
```

### Переменные окружения

Создайте файл `.env` в корне проекта:

```env
VITE_API_URL=https://your-directus-instance.com
```

## 🗄️ Структура базы данных (Directus)

### Основные коллекции

**news** - новости
- `title` - заголовок
- `content` - HTML контент
- `content_clean` - очищенный текст
- `publication_date` - дата публикации
- `source_id` - ссылка на источник
- `relevance_score` - релевантность (0-100)
- `summary_ai` - AI саммари
- `tags` - связи с тегами

**sources** - источники новостей
- `name` - название
- `rating` - надежность (0-100)
- `is_active` - активен ли в системе

**tags** - теги/категории
- `name` - название тега
- `color` - цвет для отображения

**digests** - дайджесты
- `title` - заголовок
- `digest_type` - тип (daily/weekly)
- `period_start/end` - период
- `summary_ai` - описание дайджеста

## 🗃️ Клиентское хранилище (IndexedDB)

- Сохранение голосов пользователя (лайки/дизлайки)
- Кэширование статусов источников (включен/выключен в персональной ленте)

## 📊 Особенности реализации

### Фильтрация новостей
- Исключение источников, отключенных пользователем
- Фильтрация по тегам, дате, источникам и поисковому запросу
- Пагинация на сервере

### Оптимизация
- Кэширование запросов через React Query
- Ленивая загрузка на мобильных устройствах
- Мемоизация компонентов где необходимо

### Безопасность
- Внешние ссылки открываются с `rel="noopener noreferrer"`
- Санитизация HTML контента через Directus
- Защита от XSS через `dangerouslySetInnerHTML` (только доверенный контент)
